# Human-AI Collaboration
## Status of AI Usage
AI는 다양한 지적 작업을 도와주는 도구
- Computer: Bicycle for Brain
- AI: Car or Airplane?

현대 AI는 놀라운 진보
- 책 몇권 분량의 컨텍스트를 가지고 
- 인간이 n분 걸리는 작업을 50% 이상 성공
- 필요에 따라 몇시간/며칠도 일할수 있음

하지만 대부분의 사람들은 ChatGPT 수준
1. 사람이 AI의 도움받을 영역을 판단
2. 제한된 Context + 수동으로 프롬프트 작성
3. 결과물의 일부를 AI가 생성
4. 사람이 복붙하여 결과물을 활용

이는 현재 AI 역량을 제대로 활용하지 못하는 결과

## Agentic AI의 혁명
How Claude Code changed the game
- AI works within developer's workspace
- AI can work on long-range tasks autonomously
	- Github Issues :  work items
	- Pull Requests : work records
- AI can take advange of various tools

Code vs a Knowledge Work
- 코딩은 결과물이 프로그래밍 언어인 지식 노동
- 코딩 도구를 일반 업무에 사용할 수 있음
	- LLM은 자연어 및 각종 파일 처리에 능숙

PKM+AI: Agentic AI for non-devs
1. 단일 작업의 단순화
	1. AI가 작업 영역에서 할 일을 발견
	2. AI가 작업 전체를 보고 필요한 업무 수행
	3. AI 결과물이 작업물에 바로 반영
2. 반복되는 프롬프트/워크플로 자동화
3. 여러 작업을 병렬 실행할수도 있음

- AI는 그럴듯한 답변을 내놓음
	- 시험에서 예전에는 그럴듯한 답변으로 중간은 갔지만 이제 최하점

	- 자신의 취향에 맞춰 엄격한 평가


## 1세대 vs 2세대: 패러다임의 전환

### 사용 방식의 근본적 변화

| 차원 | 1세대 (AI 이전) | 2세대 (AI+PKM) | 게임 체인저 |
|------|----------------|----------------|-------------|
| **프롬프팅** | 매번 상황 설명 반복 | 맥락 자동 인식 | 맥락 지속성 |
| **수집** | 수동 메모 → 30% 휘발 | Ambient AI → 100% 캡처 | 완전 자동화 |
| **정리** | 수동 분류 (주 2시간) | AI 자동 분류 (0분) | 시간 회복 |
| **검색** | 키워드 매칭 | 의도 이해 | 지능적 발견 |
| **창작** | 백지 앞 좌절 | AI 파트너 | 진입장벽 제거 |
| **도구** | ChatGPT 웹만 | Obsidian+Claude+MCP | 생태계 연결 |
| **학습** | 경험 휘발 | 자동 패턴 추출 | 누적 성장 |
