Part 4

From Knowledge to Goals
지식에서 목표로

지금까지 우리는 "어떻게 지식을 관리하는가"(How)에 집중했다. 이제 "왜 지식을 관리하는가"(Why)로 시야를 넓힐 시간이다.

The Perfect PKM Paradox

핵심 질문: 지식을 잘 관리하고 있는데, 왜 내 삶은 나아지지 않을까?

깨달음: PKM은 수단이지 목적이 아니다

AI4BetterMe: 목적 중심의 지식 관리

        AI4PKM                 AI4BetterMe
          ↓                         ↓
     [How: 방법론]              [Why: 목적]
          ↓                         ↓
   지식을 어떻게 관리?         왜 지식을 관리하나?
          ↓                         ↓
   수집 → 정리 → 요약          목표 → 실행 → 성장
          ↓                         ↓
      Knowledge               Better Me

AI4BetterMe = AI4PKM + Goals + Progress + Coaching

AI4BetterMe Framework

핵심 공식: Goal = Knowledge + Habits + Metrics (KHM)

1. Knowledge: 무엇을 알아야 하는지

목표 달성에는 관련 지식이 필요하다. 모든 지식이 동등하지 않다. 목표와 관련된 지식이 우선이다.

목표 지향 지식 vs. 랜덤 정보:

구분 랜덤 정보 수집 목표 지향 지식
선택 기준 "흥미롭네!" "목표에 도움 되나?"
저장 위치 Topics/분야 Goals/목표별
연결 방식 주제별 목표 → 필요 지식
활용 시점 "언젠가..." "이번 주"

2. Habits: 어떤 습관을 형성해야 하는지

지식이 있어도 행동하지 않으면 변화 없다. 세 가지 행동 유형:

Action Type Definition Tracked Via Example Goal
Knowledge Goals
(배우기)		 새로운 역량 습득 Vault + Articles "파이썬 데이터 분석 배우기"
Behavioral Goals
(하기)		 반복적 행동 실천 Vault + Calendar "주 3회 운동하기"
Status Goals
(되기)		 상태/지위 변화 Vault + Photos + Calendar "시니어 엔지니어 되기"

3. Metrics: 어떤 변화를 측정할 것인지

측정할 수 없으면 개선할 수 없다. PKM이 중앙 허브가 되는 이유:

PKM이 알고 있는 것:

  • ✅ User's actions (from journal/calendar)
  • ✅ User's knowledge (from vault)
  • ✅ User's learnings (from articles/clippings)
  • ❌ MISSING: User's current state (WhereAmI)

Where (현황) Collection Strategy:

Data Type Source Auto-Collection Purpose Update Frequency
Digital Activity Google Calendar + MCE Yes 시간 배분 추적 Real-time
Conversations Limitless Yes 학습 & 관계 인사이트 Daily
Reading Readwise Yes 지식 습득 패턴 Daily
Real World Photos + AI Analysis Semi-auto 물리적 환경 & 습관 Daily
Creation Obsidian Yes 아웃풋 & 기여도 Real-time

Step 2: Write Detailed Goal Statement

Goal Template 활용:

# Goals/2025-Q4 Senior Engineer Path.md

## Goal Definition
**What**: 2025년 12월까지 시니어 엔지니어로 승진
**Why**: 기술적 영향력 확대 + 멘토링 기회 + 커리어 성장
**Current**: Mid-level, 3년차
**Target**: Senior-level requirements 충족

## Knowledge (무엇을 알아야 하나)
### Critical Knowledge
- [x] 시니어 승진 요구사항 파악
  - Source: Career/2025-11 HR Meeting Notes
  - Key: 기술 깊이 + 리더십 + 영향력

- [ ] 시스템 아키텍처 설계 패턴
  - Gap: 현재 feature 개발만, 전체 설계 경험 無
  - Source: Book/System Design Interview
  - Action: 이번 분기 시스템 설계 문서 2개 작성

### Knowledge NOT Needed (Yet)
- ❌ Executive leadership skills (아직 시니어도 아님)
- ❌ 다른 언어/프레임워크 (현재 스택 마스터가 우선)

## Habits (어떻게 행동하나)
### Weekly Habits
- [ ] 화/목 아침: 1시간 시스템 설계 공부
- [ ] 금요일: 주간 기술 블로그 작성
- [ ] 월 1회: 팀 외부 엔지니어와 커피챗

## Metrics (무엇을 측정하나)
### Leading Indicators (과정)
- 📚 시스템 설계 공부 시간: 목표 주 2시간
- ✍️ 기술 블로그 발행: 목표 월 4개
- 🤝 외부 엔지니어 만남: 목표 월 1회

### Lagging Indicators (결과)
- 🎯 설계 문서 작성: 목표 분기 2개
- 🌟 크로스팀 프로젝트 리딩: 목표 분기 1개
- 📈 기술 블로그 조회수: 목표 총 1000+

# Goals/2025-11 Senior Engineer - Month 1 Retro.md

## What Worked
✅ **공부 루틴**: 화/목 아침이 완벽
   - Completion: 100% (4주 연속)
   - Insight: 아침이 저녁보다 일관성 ↑

✅ **커피챗**: 목표의 4배 달성
   - Result: Sarah와 크로스팀 프로젝트 시작
   - Bonus: 2명 더 멘토 연결

## What Didn't Work
⚠️ **블로그 작성**: 50% 달성 (목표 4개, 실제 2개)
   - Blocker: 금요일 저녁 = 피곤함
   - 학습: 요일보다 에너지가 중요

## Adjustments for Month 2
🔧 **Changes**:
1. 블로그: 금요일 저녁 → 수요일 오전
2. 공부: 화/목 그대로 유지 (working!)
3. 커피챗: 월 1회로 줄임 (이미 충분)

🎯 **New Targets**:
- 블로그: 4개 → 3개 (현실적으로)
- 설계 문서: 1개 완성 (우선순위 ↑)
- 크로스팀 프로젝트: 킥오프 완료

Summary: From Knowledge to Goals

핵심 변화:

Before: 지식 수집 → 정리 → 보관 → (그냥 쌓임)
After:  목표 정의 → 필요 지식 → 학습 → 실행 → 측정 → 달성

AI4BetterMe의 Promise:

  1. 목적 있는 학습: 모든 지식이 목표와 연결
  2. 실행 중심: 아는 것 → 하는 것
  3. 데이터 기반: 감이 아닌 측정
  4. AI 코칭: 혼자가 아닌 파트너와 함께
  5. 지속 가능: Burnout 아닌 성장

시작하기:

  1. 하나의 목표부터 시작
  2. Goal Template 작성
  3. 데이터 수집 설정
  4. 첫 주 실행
  5. AI와 함께 리뷰

Remember

"완벽한 시스템이 아니라 실행하는 시스템이 중요하다"

오늘 하나의 목표를 정의하고, 내일 하나의 습관을 실행하자
